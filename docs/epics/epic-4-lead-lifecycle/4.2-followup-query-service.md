# Story 4.2: Follow-up Query Service

**Epic:** 4 — Lead Lifecycle
**Status:** completed

## User Story

As a developer,
I want isolated query functions that return leads eligible for each follow-up attempt,
so that the follow-up executor (Story 4.3) can retrieve the correct leads without duplicating query logic.

## Acceptance Criteria

1. O módulo `backend/services/followup_query.py` existe e exporta duas funções públicas: `get_leads_for_first_followup(tenant_id: str) -> list[dict]` e `get_leads_for_second_followup(tenant_id: str) -> list[dict]`.
2. `get_leads_for_first_followup` retorna leads com `status IN ('new', 'qualifying')`, `last_contact_at < now() - interval '48 hours'` e `follow_up_1_sent_at IS NULL`.
3. `get_leads_for_second_followup` retorna leads com `status IN ('new', 'qualifying')` e `follow_up_1_sent_at < now() - interval '7 days'`.
4. Leads com status `qualified`, `scheduled`, `handoff` ou `cold` NUNCA aparecem no resultado de nenhuma das duas funções.
5. Ambas as funções chamam `set_tenant_context(tenant_id)` antes de qualquer query ao Supabase (NFR3).
6. Nenhum dado de outro tenant é exposto: as queries operam exclusivamente com o `tenant_id` recebido como parâmetro, e o RLS do Supabase reforça o isolamento.
7. Se `last_contact_at` for NULL em um lead, ele não aparece nos resultados de `get_leads_for_first_followup` (considerar NULL como lead que nunca teve contato — sem janela definida para follow-up; ignorar).
8. Testes unitários mockam o cliente Supabase e verificam: filtros corretos são passados para cada query; `set_tenant_context` é chamado antes da query; listas vazias são retornadas sem erro.

## Technical Notes

> O cliente Supabase é obtido via `db.client.get_client()` e `db.client.set_tenant_context()` — mesmo padrão de `webhook_service.py`.
>
> As queries de intervalo de tempo devem usar filtros Supabase com operadores `lt` e `lte`. Como o Supabase Python SDK não suporta `now() - interval` diretamente em filtros de string, calcular o threshold em Python com `datetime.now(timezone.utc) - timedelta(hours=48)` e passar como string ISO 8601.
>
> Os campos mínimos a retornar por cada lead: `id`, `tenant_id`, `phone`, `name`, `status`, `last_contact_at`, `follow_up_1_sent_at`.
