# Story 3.2: Histórico de Conversa como Contexto

**Epic:** 3 — AI Agent Core
**Status:** completed

## User Story

As a lead,
I want the assistant to remember what I said earlier in our conversation,
so that I don't have to repeat myself across messages.

## Acceptance Criteria

1. `agent_service.process_message` aceita um parâmetro `history: list[dict]` com mensagens anteriores no formato `[{"role": "user"|"assistant", "content": "..."}]`.
2. O histórico é carregado em `webhook_service.py` (ou numa camada de serviço dedicada) via query a `messages` filtrada por `tenant_id` + `phone`, ordenada por `created_at ASC`, limitada às últimas 20 mensagens — garante contexto sem overflow de tokens.
3. Antes da query de histórico, `set_tenant_context(tenant_id)` é chamado (NFR3 — RLS enforcement).
4. A resposta do agente é persistida na tabela `messages` com `direction = "outbound"`, `phone` do lead e `content` da resposta, associada ao `lead_id` se o lead já existir (ou `lead_id = NULL` se ainda não).
5. A query de carga do histórico é executada via `run_in_executor` para não bloquear o event loop (padrão estabelecido em Story 2.4).
6. O campo `role` é inferido do campo `direction`: `inbound → "user"`, `outbound → "assistant"`.
7. Testes em `backend/tests/test_agent_history.py` cobrem: histórico carregado corretamente formatado para o Claude; resposta outbound persistida; histórico vazio (primeira mensagem) funciona sem erro.

## Technical Notes

> A tabela `messages` já tem `direction`, `phone`, `content` e `created_at` — nenhuma migração de schema é necessária nesta story.
>
> Limite de 20 mensagens é suficiente para MVP; se o lead tiver histórico maior, as mais antigas são descartadas (sliding window simples, sem summarization).
>
> A persistência da resposta outbound deve ocorrer após `send_message` retornar com sucesso — se o envio falhar, não persistir (consistência simples).
>
> O histórico deve ser passado ao Claude **antes** da mensagem atual do usuário. A mensagem atual não vai no histórico passado, vai como último item `{"role": "user", "content": texto_atual}` no array `messages` do Claude.
