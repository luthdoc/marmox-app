# Story 3.1: Agente Claude Haiku — Substituição do Echo

**Epic:** 3 — AI Agent Core
**Status:** completed

## User Story

As a marble company owner,
I want my WhatsApp number to respond leads with an AI assistant instead of an echo,
so that leads receive real answers from the moment the agent goes live.

## Acceptance Criteria

1. `ANTHROPIC_API_KEY` está declarado em `backend/core/config.py` (pydantic-settings) e em `.env.example`.
2. `anthropic` está em `backend/requirements.txt` com versão fixada.
3. `backend/services/agent_service.py` expõe uma função `async def process_message(tenant_id: str, tenant_name: str, phone: str, text: str) -> str` que chama Claude Haiku (`claude-haiku-4-5-20251001`) e retorna o texto da resposta.
4. O system prompt identifica o assistente como "assistente virtual da [Nome da Empresa]" e instrui o agente a nunca fingir ser humano (FR2) — o tenant name é injetado dinamicamente por chamada.
5. O system prompt é enviado com `cache_control: {"type": "ephemeral"}` para habilitar prompt caching (NFR5 — reduz custo de LLM).
6. `backend/services/webhook_service.py` substitui a chamada a `_dispatch_echo` por `_dispatch_agent`, que chama `agent_service.process_message` e envia a resposta via `send_message` — a estrutura fire-and-forget com `asyncio.create_task` é mantida.
7. Tenants em status `onboarding` continuam sem resposta automática do agente (a verificação de status permanece no webhook_service).
8. Testes em `backend/tests/test_agent_service.py` cobrem: chamada ao Claude com prompt correto (mock da SDK); resposta retornada corretamente; falha na API retorna exceção propagada ao chamador.
9. Testes em `backend/tests/test_webhook_dispatch.py` cobrem: tenant ativo → `process_message` chamado (mock); tenant onboarding → `process_message` não chamado.
10. Antes de qualquer query a `tenants` no webhook, `set_tenant_context(tenant_id)` é chamado garantindo RLS (NFR3).

## Technical Notes

> `agent_service.process_message` recebe apenas `tenant_name` nesta story — histórico de conversa e dados do tenant serão adicionados nas Stories 3.2 e 3.4.
>
> Usar `anthropic.AsyncAnthropic` para compatibilidade com o event loop do FastAPI.
>
> Prompt caching: o bloco `system` deve ser passado como lista com `cache_control` no último bloco, conforme SDK Anthropic:
> ```python
> system=[{"type": "text", "text": SYSTEM_PROMPT, "cache_control": {"type": "ephemeral"}}]
> ```
>
> O `_dispatch_agent` em webhook_service.py deve buscar o `tenant_name` junto com `id` e `status` na query de resolução do tenant (adicionar `name` ao `.select("id, status, name")`).
