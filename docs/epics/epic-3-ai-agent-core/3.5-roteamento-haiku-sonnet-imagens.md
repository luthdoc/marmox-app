# Story 3.5: Roteamento Haiku→Sonnet e Análise de Imagens

**Epic:** 3 — AI Agent Core
**Status:** completed

## User Story

As a lead,
I want to send photos of my kitchen or countertop and have the assistant understand what I'm showing,
so that I can get more accurate help without describing everything in text.

## Acceptance Criteria

1. O webhook `POST /webhook/whatsapp` processa mensagens de imagem do Z-API (payloads com `imageMessage` no campo `message`) além das mensagens de texto já suportadas.
2. A URL da imagem é extraída do payload Z-API e armazenada em `messages.media_url` ao persistir a mensagem inbound.
3. `agent_service.process_message` aceita um parâmetro opcional `image_url: str | None` — quando presente, a mensagem é enviada ao Claude Sonnet (`claude-sonnet-4-6`) com a imagem como bloco `image` (FR4, FR12).
4. A imagem é passada ao Claude via URL (tipo `"image_url"`) sem download local — o Sonnet acessa diretamente pela URL do Z-API.
5. O roteamento automático Haiku→Sonnet ocorre também para mensagens de texto quando o conteúdo contém indicadores de complexidade: palavras-chave de reclamação (`"problema"`, `"erro"`, `"insatisfeito"`, etc.) ou intenção ambígua detectada por um classificador simples de palavras-chave (lista configurável em `agent_service.py`). FR12.
6. `agent_service.process_message` expõe o parâmetro `model: str` com default `"claude-haiku-4-5-20251001"` — o dispatcher passa `"claude-sonnet-4-6"` quando imagem ou complexidade são detectadas.
7. O schema do payload Z-API em `backend/schemas/webhook.py` é atualizado para reconhecer `imageMessage` e expor `image_url: str | None` e `is_image_message: bool`.
8. Testes em `backend/tests/test_routing.py` cobrem: payload com imageMessage → `process_message` chamado com `image_url` e model=Sonnet; texto com palavra de reclamação → model=Sonnet; texto simples → model=Haiku.
9. Testes em `backend/tests/test_image_handling.py` cobrem: imagem persistida com `media_url`; chamada ao Claude com bloco `image` correto (mock da SDK).
10. Antes de qualquer query, `set_tenant_context(tenant_id)` é chamado (NFR3).

## Technical Notes

> Z-API envia imagens com `message.imageMessage.url` ou `message.imageMessage.directPath` — verificar formato exato no payload real (ou documentação Z-API) durante implementação; adaptar o schema conforme necessário.
>
> Prompt caching não se aplica ao bloco de imagem — apenas ao system prompt. O custo de Sonnet com vision é maior; a detecção de complexidade por palavras-chave antes de subir para Sonnet é o mecanismo de contenção de custo (NFR5).
>
> A lista de palavras-chave de complexidade em `agent_service.py` deve ser uma constante Python facilmente editável — não hardcoded espalhado pelo código.
>
> Para imagens, o lead.status não é alterado automaticamente — a imagem é contexto adicional, não um passo de qualificação por si só.
