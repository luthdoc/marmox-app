# Story 5.2: Agente de Onboarding — Serviço e Parsing

**Epic:** 5 — Onboarding Agent
**Status:** completed

## User Story

As a marble company owner,
I want the WhatsApp bot to guide me through company setup in natural conversation,
so that the agent is configured without me needing to access any web interface.

## Acceptance Criteria

1. `services/onboarding_service.py` exporta `process_onboarding_message(tenant_id: str, text: str, history: list[dict]) -> str` que chama Claude Haiku com um system prompt de onboarding e retorna a resposta bruta.
2. O system prompt do agente de onboarding instrui o Claude a coletar exatamente os cinco campos: nome da empresa, serviços oferecidos (lista), regiões atendidas (lista), horário de funcionamento e mensagem de boas-vindas.
3. O agente de onboarding nunca menciona preços, prazos ou informações não fornecidas pelo operador — se não souber, pede a informação ao dono (NFR4).
4. Ao final de cada resposta, o Claude inclui obrigatoriamente o bloco `[DADOS_EMPRESA]...[/DADOS_EMPRESA]` com os dados coletados até o momento. Campos ainda não coletados aparecem como `null`.
5. O formato JSON dentro do bloco é: `{"name": <string|null>, "services": <list|null>, "regions": <list|null>, "business_hours": <string|null>, "welcome_message": <string|null>, "onboarding_complete": <bool>}`. `onboarding_complete` é `true` apenas quando todos os cinco campos forem não-nulos.
6. `services/onboarding_service.py` exporta `parse_empresa_block(raw: str) -> tuple[dict | None, str]` que extrai o JSON do bloco e retorna `(dados, texto_limpo)`. Se o bloco estiver ausente ou malformado, retorna `(None, raw)`.
7. O prompt caching está habilitado no system prompt (mesmo padrão de `agent_service.py`: `cache_control: ephemeral`).
8. Testes unitários em `backend/tests/test_onboarding_service.py` verificam: `parse_empresa_block` com resposta completa, incompleta e sem bloco; `process_onboarding_message` com mock do cliente Anthropic retornando resposta sintética.

## Technical Notes

> `process_onboarding_message` usa `_MODEL_HAIKU` de `agent_service.py` (reuse da constante já definida) e `AsyncAnthropic()` (mesmo padrão de `agent_service.process_message`).
>
> O `history` passado para `process_onboarding_message` é a conversa prévia do onboarding (mensagens armazenadas na tabela `messages` com `lead_id = NULL`). O carregamento do histórico é responsabilidade de quem chama o serviço (Story 5.3 usará `load_conversation_history` de `db/conversation.py`).
>
> O prompt deve ser direto: identificar que está configurando o assistente da marmoraria, listar os cinco campos necessários e orientar a coletar um por vez em ordem natural.
