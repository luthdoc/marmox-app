# Story 2.2: Endpoint de Webhook Z-API

**Epic:** 2 — WhatsApp Core
**Status:** completed

## User Story

As the Z-API service,
I want to POST incoming WhatsApp message events to `/webhook/whatsapp`,
so that the backend receives, validates, and persists each message associated with the correct tenant.

## Acceptance Criteria

1. `POST /webhook/whatsapp` retorna HTTP 200 com body `{"received": true}` para payloads válidos.
2. O endpoint valida a presença do header `X-Zapi-Token` — se ausente ou diferente do `ZAPI_TOKEN` configurado, retorna HTTP 401 e não persiste nada.
3. Payloads que não sejam mensagens de texto (ex: `type != "ReceivedCallback"` ou sem campo `text.message`) são ignorados silenciosamente com resposta 200 — o endpoint nunca deve retornar 4xx para payloads desconhecidos do Z-API.
4. Para mensagens de texto válidas, o serviço resolve o `tenant_id` a partir do `instanceId` no payload — busca na tabela `tenants` por `zapi_instance_id`. Se não encontrar tenant, loga warning e retorna 200 (sem persistir).
5. A mensagem é persistida em `messages` com `direction = "inbound"`, `tenant_id` correto e `lead_id` nulo (lead lookup é responsabilidade de stories futuras).
6. Toda mensagem recebida gera um log INFO estruturado com `tenant_id`, `phone`, `message_length` e `instance_id`.
7. Um teste em `backend/tests/test_webhook.py` cobre: token ausente (401), payload inválido (200), mensagem válida com tenant encontrado (200 + registro no banco mockado), tenant não encontrado (200 sem persistir).

## Technical Notes

> Payload típico do Z-API para mensagem de texto recebida:
> ```json
> {
>   "instanceId": "...",
>   "type": "ReceivedCallback",
>   "phone": "5511999999999",
>   "text": { "message": "Olá" },
>   "momment": 1234567890
> }
> ```
>
> Criar router em `backend/routers/webhook.py` e service em `backend/services/webhook_service.py`. O router só faz parse e delega ao service. O service faz lookup do tenant e persiste. Usar `AsyncClient` do `supabase-py` para operações assíncronas no contexto FastAPI.
>
> O `instanceId` no payload mapeia para `tenants.zapi_instance_id`.
