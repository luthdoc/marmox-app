# Story 1.6: Deploy do Frontend na Vercel

**Epic:** 1 — Foundation
**Status:** completed

## User Story

As a developer,
I want the Next.js frontend deployed and accessible on Vercel,
so that the production URL is established and subsequent epics can deploy incrementally via git push.

## Acceptance Criteria

1. O projeto Next.js está conectado à Vercel e faz deploy automático a cada push na branch `main`.
2. A Vercel está configurada para usar `frontend/` como Root Directory do projeto.
3. As variáveis de ambiente do `frontend/.env.example` estão configuradas no painel da Vercel.
4. `GET https://<vercel-url>/` retorna a página placeholder "Marmax" com HTTP 200.
5. O build na Vercel conclui sem erros (zero erros TypeScript e lint).
6. O domínio de preview gerado pela Vercel está acessível publicamente.

## Technical Notes

> Configurar o Root Directory como `frontend/` nas configurações do projeto na Vercel — sem isso, a Vercel tentará fazer build da raiz do monorepo e falhará. O framework preset deve ser detectado automaticamente como Next.js. Não configurar domínio customizado nesta story — o subdomínio `.vercel.app` é suficiente para o MVP inicial.
