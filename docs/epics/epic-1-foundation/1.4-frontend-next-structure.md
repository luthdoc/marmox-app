# Story 1.4: Frontend Next.js com Estrutura Mínima

**Epic:** 1 — Foundation
**Status:** completed

## User Story

As a developer,
I want a Next.js frontend with a minimal home page and the design system baseline configured,
so that subsequent epics can build product screens on top of a consistent foundation.

## Acceptance Criteria

1. `npm run dev` sobe o servidor na porta 3000 sem erros ou warnings de lint.
2. `npm run build` conclui com sucesso (zero erros TypeScript).
3. A página raiz (`/`) exibe apenas o nome da aplicação "Marmax" com fonte Inter em 17px — sem conteúdo de produto.
4. O design system baseline está configurado no Tailwind: cor `action-blue` mapeada para `#0066cc`, família de fonte `sans` apontando para Inter + system-ui.
5. O arquivo `frontend/app/layout.tsx` carrega a fonte Inter via `next/font/google` e aplica a classe ao `<body>`.
6. `npm run lint` retorna sem erros.

## Technical Notes

> Não instalar bibliotecas de componentes (shadcn, MUI, etc.) ainda — a Epic 6 decide isso. O objetivo desta story é apenas ter o Tailwind e a fonte alinhados ao design system `apple-DESIGN.md` antes de construir telas. A página raiz é um placeholder — não representa a tela de Login nem nenhuma outra tela de produto.
