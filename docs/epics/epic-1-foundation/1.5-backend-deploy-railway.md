# Story 1.5: Deploy do Backend no Railway

**Epic:** 1 — Foundation
**Status:** completed

## User Story

As a developer,
I want the FastAPI backend deployed and accessible on Railway,
so that the production URL is established and the health-check endpoint is reachable from the internet.

## Acceptance Criteria

1. O repositório contém `backend/Procfile` com o comando de start: `web: uvicorn main:app --host 0.0.0.0 --port $PORT`.
2. O repositório contém `backend/runtime.txt` especificando a versão do Python (ex: `python-3.12`).
3. O Railway está configurado para servir a pasta `backend/` como root do serviço.
4. As variáveis de ambiente do `.env.example` estão configuradas no painel do Railway (com valores reais de desenvolvimento/staging).
5. `GET https://<railway-url>/health` retorna `{"status": "ok", "version": "0.1.0"}` com HTTP 200.
6. Logs estruturados em JSON aparecem no painel de logs do Railway.

## Technical Notes

> O Railway detecta automaticamente o `Procfile` para o comando de start. Usar `$PORT` dinâmico é obrigatório — o Railway injeta a porta. Não fixar a porta em 8000. O serviço deve ser configurado para o diretório raiz `backend/` via Railway Settings > Root Directory. Não é necessário Dockerfile para esta story — o buildpack do Railway com Procfile é suficiente para o MVP.
