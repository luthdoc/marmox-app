# Story 1.3: Configuração de Variáveis de Ambiente

**Epic:** 1 — Foundation
**Status:** pending

## User Story

As a developer,
I want a `.env.example` file documenting all required environment variables,
so that any developer or deployment environment can configure the app correctly without guessing.

## Acceptance Criteria

1. `backend/.env.example` existe e documenta todas as variáveis necessárias: `SUPABASE_URL`, `SUPABASE_SERVICE_KEY`, `ANTHROPIC_API_KEY`, `ZAPI_INSTANCE_ID`, `ZAPI_TOKEN`, `APP_ENV` (valores: `development`/`production`), `LOG_LEVEL` (default: `INFO`).
2. `frontend/.env.example` existe e documenta `NEXT_PUBLIC_SUPABASE_URL`, `NEXT_PUBLIC_SUPABASE_ANON_KEY`, `NEXT_PUBLIC_APP_ENV`.
3. Existe um módulo `backend/core/config.py` que carrega as variáveis via `pydantic-settings` e falha com mensagem clara se uma variável obrigatória estiver ausente.
4. O `.gitignore` da raiz ignora arquivos `.env` e `.env.local` (mas não `.env.example`).
5. O `README.md` contém uma seção "Environment Setup" referenciando os arquivos `.env.example` de cada app.

## Technical Notes

> Usar `pydantic-settings` (v2) com `BaseSettings` para carregar e validar variáveis. Isso garante falha rápida no boot se variável obrigatória faltar, em vez de erro obscuro em runtime. Variáveis opcionais devem ter `default` declarado. `LOG_LEVEL` e `APP_ENV` são opcionais com defaults seguros.
