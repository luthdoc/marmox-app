# Story 1.2: Backend FastAPI com Health-Check e Logging

**Epic:** 1 — Foundation
**Status:** pending

## User Story

As a developer,
I want a FastAPI backend with a `/health` endpoint and structured logging configured,
so that I can verify the service is running and trace requests in production.

## Acceptance Criteria

1. `GET /health` retorna `{"status": "ok", "version": "0.1.0"}` com HTTP 200.
2. Logging estruturado em JSON está configurado globalmente — cada log contém `timestamp`, `level`, `logger` e `message`.
3. Toda requisição recebida gera um log de nível INFO com método HTTP, path e status code da resposta.
4. Erros não tratados retornam HTTP 500 com body `{"error": "internal_server_error"}` e geram log de nível ERROR com stack trace.
5. O servidor inicia com `uvicorn main:app --reload` sem warnings ou erros.
6. As variáveis de ambiente são carregadas via `python-dotenv` ao iniciar — ausência de variável opcional não quebra o boot.

## Technical Notes

> Usar `python-json-logger` para logging estruturado. Configurar um middleware ASGI (Starlette) para o request logging — não decorar cada rota individualmente. O handler de erro global deve usar o exception handler do FastAPI (`@app.exception_handler(Exception)`).
