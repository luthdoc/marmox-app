# Story 1.1: Estrutura do Monorepo

**Epic:** 1 — Foundation
**Status:** completed

## User Story

As a developer,
I want a monorepo with `backend/` and `frontend/` folders properly configured,
so that both apps can be developed, versioned, and deployed independently from a single repository.

## Acceptance Criteria

1. O repositório contém as pastas `backend/` e `frontend/` na raiz.
2. `backend/` contém `requirements.txt` com as dependências mínimas do FastAPI (fastapi, uvicorn, python-dotenv).
3. `frontend/` contém um projeto Next.js (App Router) inicializado com TypeScript e Tailwind CSS.
4. A raiz do repositório contém `.gitignore` cobrindo artefatos de Python (`__pycache__`, `.venv`, `*.pyc`) e Node (`node_modules`, `.next`).
5. A raiz contém `README.md` com instruções mínimas de setup para backend e frontend.
6. `npm run dev` no frontend e `uvicorn main:app --reload` no backend executam sem erros.

## Technical Notes

> Next.js deve ser inicializado com App Router (não Pages Router). Tailwind CSS v3 como padrão do projeto conforme design system. TypeScript habilitado. Sem src/ directory — usar estrutura plana dentro de `frontend/`.
